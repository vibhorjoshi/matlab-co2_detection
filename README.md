## 1. Data Download & Setup Instructions

Due to GitHub's strict file size constraints, the raw 41 GB AVIRIS dataset (`f250923t01p00r13_rfl.dat`) cannot be hosted directly in this repository. 

To ensure reviewers and judges can execute the code immediately without downloading massive external datasets, a pre-cropped Region of Interest (ROI) is provided.

**Setup Instructions:**
1. Clone this repository to your local machine.
2. Ensure the sample dataset `proposed_results.mat` is located in the root directory alongside the scripts. The scripts use relative pathing (`pwd`) and will automatically detect the data.
3. Run `generate_all_figures.m` to execute the full pipeline and generate the geospatial overlay.
4. Run `histogram_comparison.m` to generate the statistical performance charts.
5. All resulting figures and tables will be automatically saved to a new folder named `updated_output_file/`.

*(Optional: If you wish to run the code on the full AVIRIS scene, download the flightline from the NASA JPL AVIRIS Data Portal, place the `.dat` and `.hdr` files in a `/datasets` folder, and update the `load()` functions to use MATLAB's `hypercube` function).*

## 2. Project Folder Structure

This project has been streamlined from a fragmented, multi-file approach into two highly optimized, self-contained master scripts to guarantee reproducibility and prevent pathing errors.

*   **`generate_all_figures.m`**: The primary execution script. It loads the datacube, calculates four distinct anomaly detection algorithms (CIBR, JRGE, SFA, and the proposed CT-ACE) from scratch, and outputs all comparative visualizations and geospatial mapping.
*   **`histogram_comparison.m`**: The statistical validation script. It computes the normalized score distributions of the baseline matched filter versus the proposed framework, proving the suppression of background clutter.
*   **`proposed_results.mat`**: The standalone sample dataset containing the `cube` (hyperspectral data matrix) and `wavelengths` arrays.
*   **`updated_output_file/`**: The directory automatically generated by the scripts to store high-resolution `.png` figures and `.csv` metrics tables.

## 3. The Algorithm: Why CT-ACE?

### The Problem with Standard Matched Filters
Standard algorithms often fail on gas detection tasks because they search for positive spectral signatures within raw Reflectance ($R$) data. However, gas absorbs light, creating a *negative* spectral dip. When standard filters encounter this, they project a negative score, causing the plume to be erased when researchers clip negative noise to zero. Furthermore, global filters suffer from **scale variance**—meaning shadows, bright rooftops, and solar illumination changes trigger massive false alarms.

### Our Approach: Cluster-Tuned Adaptive Coherence Estimator (CT-ACE)
We solve these physical and mathematical constraints through a 4-step framework:

1.  **Physics-Based Preprocessing (Beer-Lambert Law):** We mathematically convert Reflectance to Absorbance using $A = -\log(R)$. In absorbance space, gas signatures become strictly positive and additive, resolving the negative-clipping bug.
2.  **Water Vapor Masking:** The opaque atmospheric water absorption window (1800 nm – 1950 nm) is explicitly sliced out of the data to prevent it from poisoning the covariance matrix.
3.  **K-Means Clustering:** The background landscape is segmented into distinct spectral classes (e.g., soil, vegetation) to prevent localized anomalies from being washed out by the global average.
4.  **Scale-Invariant Localization:** We apply the Adaptive Coherence Estimator (ACE). Unlike standard filters, ACE normalizes the projection by the local pixel's Mahalanobis distance. 

The ACE metric is defined as:
$$ D_{ACE}(x) = \frac{((x - \mu)^T C^{-1} d)^2}{((x - \mu)^T C^{-1} (x - \mu))(d^T C^{-1} d)} $$
This normalization makes the algorithm immune to surface brightness variations, ensuring it only responds to the specific CO₂ spectral shape.

## 4. Results and Evaluation

Our comprehensive evaluation proves the proposed CT-ACE framework vastly outperforms baseline continuum removal and spectral fitting techniques. 

### Selectivity Metrics (Signal-to-Background Ratio)
Visualizations can be subjective, but the Signal-to-Background Ratio (SBR) provides undeniable mathematical proof of algorithm selectivity. As shown in **Table 1**, baseline algorithms struggle to differentiate the gas from the background, yielding SBRs near 1.0 to 2.6. The proposed CT-ACE method achieves an SBR of **45.5**, proving it aggressively suppresses background clutter while preserving the target.

| Algorithm | Mean Target Score | Mean Background Score | SBR Selectivity |
| :--- | :--- | :--- | :--- |
| CIBR | 0.10088 | 0.038296 | 2.6343 |
| JRGE | 0.24659 | 0.13761 | 1.7920 |
| SFA | 0.10522 | 0.11319 | 0.9296 |
| **CT-ACE (Proposed)** | **0.24416** | **0.00536** | **45.5020** |

### Visual and Statistical Proof

*   **Figure 1 (Stagewise Ablation):** Demonstrates the visual progression of the pipeline. While CIBR and SFA leave heavy striping and background artifacts, CT-ACE completely isolates the cohesive plume structure.
*   **Figure 2 (1D Profile Analysis):** A cross-sectional slice through the image array. The baseline methods are chaotic across the entire spatial axis. The CT-ACE response remains completely flat at `0.0` until the exact spatial index of the gas plume, illustrating a pristine noise floor.
*   **Figure 3 (Threshold Sensitivity):** Proves the algorithm's resistance to false alarms. The baseline CIBR confidence is smeared across background noise, meaning a lower threshold yields thousands of false pixels. The proposed algorithm's response stays near zero until the 95th percentile, guaranteeing high confidence detections.
*   **Figure 4 (Geospatial Validation):** Utilizing the MATLAB Mapping Toolbox (`geoaxes`, `projinv`), the P95 hotspot mask is accurately projected from UTM coordinates into Latitude/Longitude, overlaying the exact anomaly onto a real-world satellite basemap for practical environmental monitoring.
